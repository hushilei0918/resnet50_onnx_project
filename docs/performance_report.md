# ResNet50 推理优化性能测试报告

## 1. 测试环境
- CPU: Intel i5-10400 6核
- 内存: 16GB DDR4
- 系统: Ubuntu 22.04
- PyTorch 版本: 2.4.0
- ONNX Runtime 版本: 1.19.2
- 测试模型: ResNet50（ImageNet 预训练）

## 2. 测试方案
- 预热轮数: 10轮
- 正式测试轮数: 200轮
- 测试输入: 1x3x224x224 随机张量
- 对比维度：平均延迟、P95延迟、吞吐量、内存占用

## 3. 测试结果
【性能对比汇总】
指标              PyTorch      ONNX Runtime    提升倍数      
------------------------------------------------------------
平均延迟(ms)        43.35        9.37            4.63      x
P95延迟(ms)       49.75        10.62           4.68      x
吞吐量(FPS)        23.07        106.68          4.62      x
内存占用(MB)        121.39       182.57          0.66      x

## 4. 优化手段
1. 将 PyTorch 动态图模型导出为 ONNX 静态图格式
2. 开启 ONNX Runtime 全量图优化（常量折叠、算子融合等）
3. 配置最优 CPU 线程数，提升并行效率

## 5. 结论
通过 ONNX Runtime 优化，ResNet50 图像分类服务的推理吞吐量提升 4.62 倍，延迟降低 4.6 倍，在相同硬件条件下可承载更多并发请求，显著降低部署成本。
